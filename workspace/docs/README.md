# Docs

Project folders for specific work.

---

## Naming Convention

Use dated folders for easy organization:

```
docs/
├── 2024-01-15 - Website Redesign/
├── 2024-01-20 - Client Proposal/
├── 2024-02-01 - Product Launch/
└── ...
```

---

## What Goes Here

- **Client projects** - Deliverables, notes, assets
- **Internal projects** - Research, planning, documentation
- **One-off work** - Things that don't fit elsewhere

---

## Folder Structure

Each project folder might contain:
```
2024-01-15 - Website Redesign/
├── brief.md          # Project requirements
├── research.md       # Background research
├── drafts/           # Work in progress
└── final/            # Completed deliverables
```

---

## Get Started

Create a folder when you start a new project. The date prefix keeps things chronological.
